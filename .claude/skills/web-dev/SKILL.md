# Web Developer Skill

## When to Use

Load this skill when:
- Creating new web pages or websites
- Building frontend components with HTML/CSS/JS
- Setting up web projects from scratch
- Working with APIs and managing state
- Implementing responsive designs and animations

## Instructions

### Project Setup

To set up a new web project:

1. Create project directory structure:
   - `index.html` - Main HTML file
   - `css/` - Stylesheets directory
   - `js/` - JavaScript files directory
   - `images/` - Assets directory
2. Create basic HTML5 boilerplate with meta tags for viewport and charset
3. Link CSS and JS files properly

### Creating Components

To create web components:

1. Write semantic HTML structure using appropriate tags (header, nav, main, section, article, footer)
2. Use BEM naming convention for CSS classes (block__element--modifier)
3. Keep components modular and reusable
4. Ensure accessibility with proper ARIA attributes and semantic markup

### Styling

To implement styles:

1. Use modern CSS features (Flexbox, Grid, CSS Variables)
2. Implement responsive design with media queries
3. Follow mobile-first approach
4. Use CSS reset or normalize.css for consistent cross-browser styling
5. Optimize CSS for performance (minimize repaints, use will-change strategically)

### State and APIs

To handle state and API calls:

1. Use modern fetch API for HTTP requests
2. Implement async/await for asynchronous operations
3. Handle errors with try/catch and provide user feedback
4. Store state in memory or use localStorage for persistence
5. Follow RESTful API conventions
6. Implement debouncing for search inputs

### Best Practices

To ensure code quality:

1. Write clean, readable code with meaningful variable and function names
2. Use ES6+ features (const/let, arrow functions, destructuring, modules)
3. Comment complex logic but avoid obvious comments
4. Use version control for all projects
5. Test across browsers and devices
6. Optimize images and assets for web performance
